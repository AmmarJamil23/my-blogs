Flight Reservation System – C++ Project Using Data Structures

This project is a simple and complete flight reservation system built using C++. It is designed as part of a university Data Structures and Algorithms (DSA) course. The system helps users book, cancel, and manage flight seat reservations. It also saves the data into a file so that bookings are not lost when the program is closed.

Introduction

Airlines need proper systems to handle flight seat bookings. This project allows users to perform all basic operations like booking a seat, canceling a seat, checking available seats, and searching for a passenger. The system is menu-driven and runs in the terminal. The code uses important C++ data structures like maps, sets, and binary search trees to manage the data efficiently.

Project Objectives

  • Allow users to book and cancel flight seats.
  • Make it easy to update or search for passenger information.
  • Display available and booked seats quickly.
  • Store all booking data in a file to keep it safe even after the program is closed.
  • Use C++ data structures to make the system fast and organized.

System Design

1. Modules

The system is divided into different parts (called modules):

  • User Interface: Shows the menu and asks the user what they want to do.
  • Booking Module: Manages seat bookings and cancellations.
  • Passenger Data: Stores and sorts passengers using a binary search tree.
  • File Storage: Saves and loads booking data from a text file.

2. Main Features

  • Book a seat for a passenger.
  • Cancel a seat by seat number or passenger name.
  • Search for a passenger by name.
  • Update passenger name for any seat.
  • Display total booked seats.
  • Sort passengers alphabetically by name.
  • List all available or booked seats.
  • Show the seat with highest or lowest number booked.
  • Save and load all data from a file called flight_data.txt.

How the Code Works

Passenger Class

This class stores the name and seat number of each passenger.


class Passenger {
public:
  string name;
  int seatNumber;
  Passenger() : name(""), seatNumber(-1) {}
  Passenger(string name, int seatNumber) : name(name), seatNumber(seatNumber) {}
};

Binary Search Tree (BST)

Passengers are added to a binary search tree based on their seat numbers. This helps when we want to display booked seats in order.

Main System Class

The FlightReservationSystem class contains all the functions. It uses:

  • set to store available seat numbers.
  • unordered_map to map seat numbers to passenger details.
  • BST to store passengers in a sorted way by seat number.

Example: Booking a Seat

When the user books a seat, the system picks the lowest available seat number, assigns it to the passenger, and adds it to the BST and the map.


void bookSeat(string name) {
  if (availableSeats.empty()) {
    cout << "No seats available." << endl;
    return;
  }
  int seatNumber = *availableSeats.begin();
  availableSeats.erase(seatNumber);
  Passenger p(name, seatNumber);
  seatMap[seatNumber] = p;
  root = insertNode(root, p);
  cout << "Seat booked: " << seatNumber << endl;
}

Data Storage

All passenger data is saved to a file named flight_data.txt. When the program starts, it reads this file and loads all bookings. When the program exits, it saves all current bookings to the file. This makes sure that bookings are not lost even if the program is closed.

Testing

The system was tested by booking, canceling, updating, and searching passengers. Edge cases like canceling an unbooked seat or entering wrong input were also tested. The program handled these situations without crashing.

Limitations

  • The system is designed for single-user use. It does not support multiple users at the same time.
  • There is no graphical interface. It runs in the command-line only.
  • There is no support for managing multiple flights, only one flight with fixed seats.

Learning Outcome

This project helped us learn how to use different data structures in real-world problems. We used sets to manage available seats, maps for quick lookups, and binary search trees for sorted data. We also practiced file handling to save and load data. The project improved our understanding of C++, classes, and logic building.
