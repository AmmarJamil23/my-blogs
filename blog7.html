Understanding self in Python – Object-Oriented Programming Concept

This article explains the meaning and usage of the keyword self in Python classes. If you are learning object-oriented programming in Python, you will often see self used in class methods. Many beginners find it confusing, but this guide will explain it step by step in very simple language.

Introduction

In Python, everything is an object. When we create a class, we use it to build objects that can store data and perform actions. The keyword self is used inside class methods to refer to the object itself. It is not a special keyword in Python, just a name used by convention.

Why is self Needed?

  • self helps us refer to the object that is calling the method.
  • It allows us to store and access data that belongs to a specific object.
  • Each object has its own copy of variables, and self is how we work with them.

How to Use self

Let’s understand with a simple class:


class Student:
    def __init__(self, name, age):
        self.name = name
        self.age = age

    def show_info(self):
        print("Name:", self.name)
        print("Age:", self.age)

s1 = Student("Ali", 20)
s1.show_info()

Here is what happens:

  • The __init__ method is like a constructor. It runs when the object is created.
  • self.name means “this object's name”. It stores the value passed to name.
  • self.age stores the age for that specific object.
  • When s1.show_info() is called, self refers to s1.

Is self a Keyword?

No, self is not a keyword in Python. It is just a common name. You can technically use any name instead of self, but it is strongly recommended to use self because it makes your code more readable and standard.

What Happens If You Don’t Use self?

If you don’t use self, Python will not know where to store or access instance variables. Your program will show an error like “name is not defined” or “missing 1 required positional argument”.


class Student:
    def __init__(name, age):
        name = name
        age = age

s1 = Student("Ali", 20)
# This will not work properly, because self is missing.

Using self in Multiple Objects

Each object has its own values. Let’s see what happens when we create two students:


s1 = Student("Ali", 20)
s2 = Student("Sara", 21)

s1.show_info()
s2.show_info()

The output will be:


Name: Ali
Age: 20
Name: Sara
Age: 21

This is because self refers to s1 in the first call, and to s2 in the second call. So, each object works with its own data.

Learning Outcome

By understanding self in Python, we can write clean and reusable object-oriented code. We now know:

  • self refers to the current object.
  • It helps store and access data for that object.
  • It must be the first parameter in all instance methods.
  • Using self is a good coding practice in Python classes.

Conclusion

self is a simple but very important concept in Python. It is used in almost every class we write. Once you understand that self is just a reference to the object calling the method, it becomes much easier to write class-based code. Practice using self in your own classes to get more comfortable with object-oriented programming in Python.
